# Overlay Prototype

这个目录用来验证一件事：当前生成出的 3/4 视角独立服装层，能不能通过少量锚点对位，贴到基础模特上。

## 目标

当前不追求生产级自动拼接，只验证最小闭环：

1. 基础模特作为底层。
2. 短袖和长裤各自作为独立服装层。
3. 用多锚点三角网格把独立服装层分片仿射变换到模特上。
4. 看角度、衣长、腰线和肩线是否大体贴合。

## 当前资产

1. 基础模特：`../../refs/style/base-mannequin-best.jpeg`
2. 短袖中性结构层：`../../refs/style/short-sleeve-neutral-layer-best.jpeg`
3. 长裤中性结构层：`../../refs/style/trousers-neutral-layer-best.jpeg`

这些文件都是当前阶段冻结下来的稳定引用路径，不再直接依赖具体 run 目录。当前默认已经切到“pose-locked neutral layer”资产，而不是更早那批 garment-only 商品图资产。

## 锚点规则

原型里所有锚点都使用归一化坐标，即：

1. `x` 和 `y` 都是 `0` 到 `1` 之间的小数。
2. `0, 0` 表示图片左上角。
3. `1, 1` 表示图片右下角。
4. 页面运行时会自动乘上图片实际像素尺寸。

### 短袖网格锚点

短袖当前使用 8 个网格锚点：

1. `leftSleeveTip`
2. `leftCollar`
3. `rightCollar`
4. `rightSleeveTip`
5. `leftUnderarm`
6. `rightUnderarm`
7. `leftHem`
8. `rightHem`

作用：

1. 同时控制领口、肩线、袖子展开方向和躯干收腰。
2. 让上衣不再只能整体缩放，而是可以按多个三角面片分别贴合。
3. 验证 shape-first 的中性结构层是否值得继续做更细局部 warp。

### 长裤网格锚点

长裤当前使用 9 个网格锚点：

1. `leftWaist`
2. `rightWaist`
3. `leftHip`
4. `rightHip`
5. `crotchTop`
6. `leftKnee`
7. `rightKnee`
8. `leftHem`
9. `rightHem`

作用：

1. 把腰头、骨盆、裆部、膝点和裤脚拆开控制。
2. 让裤装不再只是一整块全局变形，而是按骨盆和腿部的局部关系贴合。
3. 验证长裤 neutral-layer 是否比早期 garment-only 输入更适合作为底稿。

## 蒙版逻辑

因为当前服装层本身就是浅色，简单按“白底阈值”去抠图会把衣身也一起吃掉，所以页面改成了更保守的最小方案：

1. 先给每件衣物定义一套手工 silhouette 外轮廓，只保留衣物主体区域。
2. 再对领口内腔、袖口开口、侧腰内腔、裤脚开衩这类明确应透明的区域，用手工 cutout 掏空。
3. 最后再做多锚点三角网格仿射对位。

这不是生产级抠图，只是为了验证“独立服装层 + 对位”路线是否可行。

## 控色逻辑

这个原型现在已经不只是“整件单色染色”，而是把颜色控制进一步拆成了两层：

1. 主色：覆盖整件衣物主体区域。
2. 辅色：只覆盖手工定义的局部区域。

当前示例里：

1. 短袖辅色作用在袖子和下摆。
2. 长裤辅色作用在腰头和侧片。

这样做的目的不是追求最终美术质量，而是验证一个更关键的产品结论：

1. 模型负责形体、褶皱、透视和贴合关系。
2. 程序负责最终展示色。
3. 如果将来需要多色款，也应优先走“区域蒙版 + 程序控色”，而不是重新让模型生成一张新颜色的衣服图。

## 当前判断

切到 neutral-layer 资产后，原型已经可以验证出一个更稳定的结论：

1. 这类贴姿态的中性结构层，确实比早期独立商品图资产更适合作为贴合输入。
2. 但即便升级到多锚点三角网格，当前结果仍然只能说明方向成立，还达不到真实穿着贴合。
3. 这说明下一步的瓶颈已经从“生成策略错误”转成“局部 warp 还不够细，而且源结构层本身还需要继续稳定”。

## 结果怎么判断

如果下面几项同时成立，就说明这条路线继续做是值得的：

1. 短袖肩线和下摆大体贴住基础模特。
2. 长裤腰头和裤脚大体贴住基础模特。
3. 两件服装不会明显错层到完全不能看。
4. 误差只剩局部扭曲，而不是整体角度错位。

如果误差主要集中在局部边界，而不是整件衣服方向错了，下一步就不该再烧 prompt，而是进入更精细的对位/warp 方案。
